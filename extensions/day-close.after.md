## Спринт-чеклист (интерактивный)

> ОБЯЗАТЕЛЬНО. Спрашивать пользователя по каждому пункту: done / skip.

1. Прочитать текущие привычки из `DS-strategy/inbox/WP-16-sprint-tracker.md` (секция с ежедневными привычками)
2. Показать пронумерованный список привычек
3. Спросить пользователя: «По каждому пункту: done / skip?»
4. Дождаться ответа
5. Записать результат в секцию «Спринт (DD мес)» итогов дня:
   - `done` / `skip` по каждой привычке
   - Итого: N/M done (X%)
   - Краткий комментарий: что улучшилось, что отстаёт

**Формат в DayPlan:**

```markdown
### Спринт (DD мес)

- done/skip CODE Название
- ...

**Спринт: N/M done (X%).** Комментарий.
```

## Аудиорефлексии (автоматически)

Запустить обработку аудиорефлексий за сегодня (если есть новые записи):

```bash
bash extensions/process-reflections.sh
```

Если новых записей нет — скрипт тихо завершится. Не блокирует Day Close.

## Рефлексия дня

> Условие: `params.yaml → reflection_enabled: true`. Иначе — пропустить.

Задать пользователю (после итогов, перед верификацией):

1. **Что сегодня было самым сложным?** Где пришлось применить усилие, а не инерцию.
2. **Что бы я сделал иначе?** Конкретное действие, не абстрактное намерение.
3. **За что себя похвалить?** Результат или решение, которым доволен.

Ответы записать в секцию «Рефлексия» итогов дня (DayPlan).
