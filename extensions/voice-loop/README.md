# Голосовой цикл IWE (WP-35 Ф1)

Hands-free voice-to-voice с Claude Code: говоришь команду → распознаётся → `claude -p`
исполняет → ответ озвучивается → цикл ждёт следующую команду. Сценарии: кухня, машина, сад.

## Запуск

```bash
bash DS-exocortex/extensions/voice-loop/voice-loop.sh
```

Обращайся по имени: **«Компьютер, …»** (как «эй Siri»). Можно одной фразой («Компьютер,
прочитай план») или просто «Компьютер» → «Да?» → команда следующей фразой. Имя должно
стоять первым словом; речь без имени игнорируется. («Клод» whisper плохо слышит —
поэтому позывной русский.)
Скажи **«стоп» / «выход» / «хватит»** или нажми Ctrl-C для завершения.

## Как устроено

```
микрофон → ffmpeg (PulseServer) → VAD по энергии → faster-whisper (STT, локально)
  → ClaudeSDKClient (живая рабочая сессия с инструментами, потоковый ход)
  → озвучка действий по ходу + финальный ответ по фразам
  → очередь озвучки → piper (TTS, локально) → ffmpeg→pulse
```

Это голосовой пульт («remote-control») к настоящей сессии Claude Code: голосом даёшь
команду → сессия выполняет (с инструментами, полным контекстом IWE) и проговаривает
действия по ходу → итог голосом → ждёт следующую команду.

- **STT и TTS локальные** → переиспользуются в офлайн-фазе (Ойтал); сейчас мозг онлайн.
- **Живая сессия (ключ к скорости):** мозг держится одним процессом через `ClaudeSDKClient`.
  Контекст IWE грузится ОДИН раз при старте (прогрев ~30с), дальше круг ~3-6с. Холодный
  `claude -p` стартовал бы ~20с КАЖДЫЙ раз — поэтому от него отказались.
- **Потоковая озвучка (ключ к отзывчивости):** ответ идёт по токенам
  (`include_partial_messages`); как только готова первая фраза — она сразу уходит в
  озвучку, пока хвост ещё генерится. На длинном ответе (список) первый звук слышен через
  ~5-7с вместо ~40с. До этого цикл ждал весь текст, потом синтезировал весь — и только
  потом говорил.
- **Озвучка действий по ходу (remote-control):** это полноценная рабочая сессия агента с
  инструментами (читает/правит файлы, запускает команды). Когда агент запускает
  инструмент, цикл проговаривает короткой фразой что он делает («читаю файл», «выполняю
  команду», «правлю файл») и печатает деталь в окно. Поэтому первый звук слышен через
  ~5с даже на медленном ходе — ты слышишь работу, а не немую паузу. Финальный итог
  озвучивается в конце.
- Сессия запускается в `~/IWE`, поэтому видит CLAUDE.md и контекст РП.
- Воспроизведение через `ffmpeg→pulse` (не ffplay: тот идёт в ALSA, которого в WSL нет).

## Настройки (env)

| Переменная | По умолчанию | Что |
|---|---|---|
| `VOICE_WHISPER_MODEL` | `small` | модель распознавания (`small`/`medium` — точнее, медленнее) |
| `VOICE_LANG` | `ru` | язык распознавания |
| `VOICE_PIPER_MODEL` | `voices/ru_RU-dmitri-medium.onnx` | голос озвучки |
| `VOICE_END_SILENCE_MS` | `900` | пауза, после которой фраза считается законченной |
| `VOICE_START_FACTOR` | `3.5` | чувствительность старта (выше = строже к шуму) |
| `VOICE_CLAUDE_MODEL` | `haiku` | модель мозга (`haiku`/`sonnet`/`opus`) |
| `VOICE_CLAUDE_PERM` | `bypassPermissions` | режим прав агента (hands-free → без запросов) |
| `VOICE_EXTRA_DIRS` | `…/Vault1` | папки вне рабочей, доступные агенту (через `:`) |
| `VOICE_WAKE` | `компьютер,…` | wake-слова через `,` (как «эй Siri»); `off` = всегда слушать |
| `VOICE_CPU_THREADS` | `8` | потоки CPU для распознавания |
| `VOICE_PERF` | `1` | печатать тайминги (распознавание / до первого звука / весь ответ) |

## Зависимости

- `ffmpeg` (запись + воспроизведение через pulse; есть в системе)
- `faster-whisper`, `piper-tts`, `claude-agent-sdk` (pip --user)
- голос piper: `python3 -m piper.download_voices ru_RU-dmitri-medium --download-dir voices`

## Известные ограничения (Ф1 MVP)

- Озвучка действий — короткий глагол по типу инструмента («читаю файл», «выполняю команду»), без подробностей вслух (деталь печатается в окно). Этого достаточно для «понимаю, что он работает».
- Точность распознавания русского на `small` средняя — настраивается моделью.
- Прерывание ответа голосом («перебить») пока нет.
- Офлайн-мозг (локальная LLM вместо `claude -p`) — отдельная фаза WP-35.
