# Obsidian Scan (headless)

Сканирует daily notes из Obsidian vault и записывает находки в inbox DS-strategy.

## Конфигурация

- **Vault:** `/mnt/c/Users/Timur/Documents/Vault1/`
- **Daily notes:** `Calendar/YYYY-MM-DD.md`
- **Период:** 3 дня (вчера, позавчера, позапозавчера)

## Алгоритм

### Шаг 1. Собрать daily notes

1. Определи период: сегодня минус 3 дня
2. Прочитай все `Calendar/YYYY-MM-DD.md` за период
3. **Фильтрация шума:** Игнорируй:
   - Блоки `` ```dataview `` и `` ```dataviewjs `` (до закрывающих `` ``` ``)
   - Блоки `> [!info]-` (callout с dataview)
   - Шаблонные секции без заполнения (пустые `- [ ]`, незаполненные промпты)
   - Ссылки на вчера `[[YYYY-MM-DD|вчера]]`
4. Извлеки содержимое из секций:
   - **«Возвращаемся во вчера»** — события и инсайты (если заполнены)
   - **«Что надо точно сделать сегодня»** — задачи
   - **«Log (дела)»** — основной контент: что делал, с кем говорил, что понял
   - **«Scratchpad»** — свободные мысли

### Шаг 2. Классифицировать находки

| Тип | Признак |
|-----|---------|
| **НЭП** | Жалоба, недовольство, «не нравится», «бесит», «надо бы» |
| **Апдейт по проекту** | Упоминание проекта (stomresearch, kense, Геонлайн, meal prep, СМ) |
| **Инсайт / знание** | Различение, паттерн, осознание, фидбек от эксперта |
| **Задача** | Конкретное действие, которое нужно сделать |
| **Шум** | Рутина без информационной ценности |

### Шаг 3. Контекстная сверка

Перед записью:
1. Прочитай текущий `DS-strategy/docs/Dissatisfactions.md` — не дублировать существующие НЭП
2. Прочитай текущий `WeekPlan` в `DS-strategy/current/` — привязать апдейты к активным РП
3. Прочитай `DS-strategy/inbox/fleeting-notes.md` — не дублировать уже записанное

### Шаг 4. Запись в inbox

**Режим: автоматический (headless).** Записывать находки без ожидания одобрения.

Для каждой находки (кроме шума):

| Тип | Куда | Действие |
|-----|------|----------|
| НЭП (новая) | `DS-strategy/docs/Dissatisfactions.md` | Добавить строку в «Активные» |
| НЭП (существующая обновлена) | `Dissatisfactions.md` | Обновить комментарий |
| Апдейт по проекту | `DS-strategy/inbox/WP-{N}-*.md` | Обновить «Текущее состояние» (если файл есть) |
| Инсайт / знание | `DS-strategy/inbox/fleeting-notes.md` | Добавить с меткой `[obsidian-scan YYYY-MM-DD]` |
| Задача | `DS-strategy/inbox/fleeting-notes.md` | Добавить с меткой `[obsidian-scan YYYY-MM-DD]` |

### Шаг 5. Лог

Вывести в stdout (попадёт в лог scheduler):

```
obsidian-scan: просмотрено N daily notes за YYYY-MM-DD — YYYY-MM-DD
  НЭП: N (новых: N, обновлённых: N)
  Апдейты: N
  Инсайты: N
  Задачи: N
  Шум: N (пропущено)
```

## Что НЕ делать

- Не модифицировать файлы в Obsidian vault (read-only)
- Не дублировать уже существующие НЭП или записи в fleeting-notes
- Не создавать РП (только предлагать через запись в fleeting-notes)
- Не извлекать dataview-скрипты как контент
