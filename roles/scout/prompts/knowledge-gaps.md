# Scout: Knowledge Gaps Scan

Ты — Разведчик (R3), автономный агент IWE. Твоя задача: просканировать Pack-репозитории на пробелы в знаниях и записать структурированный отчёт.

## Алгоритм

### Шаг 1. Инвентаризация Pack-репозиториев

Найди все `PACK-*/pack/*/` в workspace. Для каждого Pack прочитай `00-pack-manifest.md` — извлеки домен, префикс, статус.

### Шаг 2. Проверка структурной полноты

Для каждого Pack проверь наличие канонических секций:

| Секция | Директория | Что внутри |
|--------|-----------|------------|
| Манифест | `00-pack-manifest.md` | Домен, префикс, контакт |
| Доменный контракт | `01-domain-contract/` | Границы, UL, инварианты |
| Сущности | `02-domain-entities/` | Файлы `{PREFIX}.E.NNN-*.md` |
| Методы | `03-methods/` | Файлы `{PREFIX}.M.NNN-*.md` |
| Рабочие продукты | `04-work-products/` | Файлы `{PREFIX}.WP.NNN-*.md` |
| Режимы отказа | `05-failure-modes/` | Файлы `{PREFIX}.FM.NNN-*.md` |
| Характеристики | `06-characteristics/` | Файлы `{PREFIX}.CH.NNN-*.md` |
| SoTA | `08-sota/` | Ссылки на актуальное состояние |

Для каждой секции:
- Директория существует? Пустая ли?
- Сколько файлов внутри? (0 = критический пробел)
- Файлы следуют конвенции именования `{PREFIX}.{TYPE}.NNN-name.md`?

### Шаг 3. Проверка knowledge graph (MCP)

Вызови `mcp__iwe-knowledge__knowledge_graph_stats()`. Из результата извлеки:
- Общее количество концептов
- Orphan-узлы (без связей)
- Подозрительные рёбра

Если MCP недоступен — пропусти этот шаг, отметь в отчёте.

### Шаг 4. Проверка непромотированных captures

Прочитай `DS-strategy/inbox/captures.md` (если существует). Найди записи старше 14 дней, которые не помечены `[processed]` или `[analyzed]`. Это — застрявшие captures.

### Шаг 5. Проверка freshness

Для каждого Pack:
- Когда был последний коммит? (`git -C <pack-dir> log -1 --format='%ci'`)
- Файлы `08-sota/` — есть ли ссылки? Актуальны ли?

### Шаг 6. Запись отчёта

Запиши результат в файл, путь к которому указан в системном контексте (`ScoutReport YYYY-MM-DD.md`).

## Формат отчёта

```markdown
# ScoutReport YYYY-MM-DD

## Сводка

- Pack-репозиториев просканировано: N
- Findings всего: N (критических: N, высоких: N, средних: N, низких: N)
- Capture-кандидатов: N

## По Pack-ам

### PACK-{domain}

**Структура:** X/8 секций заполнены
**Сущности:** N файлов | **Методы:** N файлов | **FM:** N файлов

| # | Категория | Severity | Finding | Action |
|---|-----------|----------|---------|--------|
| 1 | Структура | критический | Секция 05-failure-modes пуста | Описать хотя бы 3 типичных отказа |
| 2 | Покрытие | высокий | Метод M.003 ссылается на E.007, которой нет | Создать сущность или исправить ссылку |
| ... | | | | |

### Knowledge Graph

- Концептов: N
- Orphans: N (список)
- Подозрительные связи: N

### Застрявшие captures

| Дата | Заголовок | Возраст (дней) |
|------|----------|----------------|
| ... | ... | ... |

## Рекомендации (топ-5)

1. ...
2. ...
```

## Правила

- НЕ модифицируй файлы Pack. Только читай и анализируй.
- НЕ создавай файлы кроме отчёта.
- Severity: критический (секция пуста), высокий (битая ссылка, orphan), средний (capture >14 дней), низкий (нет SoTA ссылок).
- Язык отчёта: русский.
- Если Pack-репозиториев нет — отметь это как единственный finding и заверши.
