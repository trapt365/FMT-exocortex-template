## Проблема

ИИ-ассистенты умеют генерировать текст, код и ответы. Но у большинства пользователей одни и те же проблемы:

- **Контекст теряется.** Каждая новая сессия с ИИ — с чистого листа. Вчерашние решения, планы, договорённости — забыты.
- **Знания остаются в голове.** Ты прошёл курс, прочитал книгу, решил задачу — но через месяц не можешь воспроизвести ход мысли.
- **ИИ заменяет мышление, а не усиливает.** Ты получаешь ответ, но не становишься компетентнее. Без ИИ — снова на нуле.
- **Нет системы.** Планы — в заметках, задачи — в голове, знания — в чатах. Всё разрозненно.
- **Время утекает.** Непонятно, над чем работал, что сделал, куда двигаешься.

## Решение: IDE, но для мышления

**IWE (Intellectual Work Environment)** — интеллектуальная рабочая среда.

Как IDE объединяет редактор, компилятор и дебаггер в одну среду для программиста — так IWE объединяет знания, планирование и ИИ-агентов в одну среду для мышления.

| IDE (для кода) | IWE (для мышления) |
|---------------|-------------------|
| Редактор → пишешь код | Экзокортекс → фиксируешь знания |
| Компилятор → проверяет синтаксис | Принципы → проверяют корректность решений |
| Дебаггер → находит ошибки | Протоколы ОРЗ (Открытие→Работа→Закрытие) → находят потери знаний и контекста |
| Линтер → улучшает качество | АрхГейт → оценивает архитектурные решения |
| Git → история изменений | Стратег → история и планирование работы |

> **Ключевой принцип: экзоскелет, не протез.** IWE усиливает ваше мышление, а не заменяет его. После каждой сессии вы становитесь компетентнее, а не просто получаете результат. Подробнее: [principles-vs-skills.md](docs/principles-vs-skills.md).

## Культура работы — новый стиль взаимодействия с ИИ

IWE — это не набор промптов. Это **культура работы**: 14 элементов (протоколы, навыки, форматы), которые превращают хаотичное взаимодействие с ИИ в управляемый процесс. Подробнее: [DP.M.008 — Операционные правила IWE](https://github.com/TserenTserenov/PACK-digital-platform/blob/main/pack/digital-platform/03-methods/DP.M.008-iwe-operating-rules.md).

### Протокол ОРЗ (Открытие → Работа → Закрытие)

Каждая сессия и каждый день проходят через три стадии:

- **Открытие** — Claude проверяет план, определяет задачу, согласовывает подход. Вы не начинаете работу «с чистого листа» — ИИ знает контекст.
- **Работа** — по ходу работы Claude фиксирует ценные знания (Capture-to-Pack). Инсайты не теряются.
- **Закрытие** — результат зафиксирован, план обновлён, следующая сессия начнётся с того, где вы остановились.

Пропуск Открытия = незапланированная работа. Пропуск Закрытия = потерянный результат.

### Экзокортекс — внешняя память

Ваши знания, принципы, различения, планы и контекст хранятся в файлах, которые Claude читает в каждой сессии. Это не «промпт» — это **накопленная база**, которая растёт вместе с вами.

### Формализация знаний (Pack)

То, что вы узнали, не остаётся в голове. Ценные знания формализуются в Pack — паспорт предметной области. Pack — единственный source-of-truth для доменного знания. Подробнее: [LEARNING-PATH.md](docs/LEARNING-PATH.md).

## Для кого

Каждый профессионал тонет в информации: 12+ инструментов (Notion, Google Docs, Slack, ChatGPT, курсы...), знания размазаны, ничего не связано. AI отвечает на вопросы, но не знает *ваш* контекст — каждый раз с нуля.

IWE — для тех, кто хочет это изменить:

- **Предприниматели и руководители** — стратегируете, принимаете решения, управляете проектами. IWE даёт систему: от еженедельного планирования до формализации знаний предметной области
- **Инженеры и разработчики** — работаете с кодом и архитектурой. IWE сохраняет контекст между сессиями, ИИ знает вашу кодовую базу, техдолг, роадмап
- **Исследователи и аналитики** — изучаете, синтезируете, публикуете. IWE превращает разрозненные заметки в структурированную базу знаний, которая растёт вместе с вами
- **Все, кто ведёт интеллектуальную работу** — и хочет **симбиоз с ИИ**, а не зависимость от него. Экзоскелет для мышления, не протез

## Как это выглядит на практике

- Утром — Стратег составил план: уведомление в Telegram + файл DayPlan в репозитории
- Открываешь VS Code → `claude` → Claude знает, что в плане, и предлагает начать с приоритетного
- Работаешь — Claude фиксирует знания по ходу (Capture-to-Pack)
- Закрываешь сессию — результат зафиксирован, план обновлён
- В понедельник — Стратег готовит черновик недельного плана, вы обсуждаете его на сессии стратегирования

## Начать

**Быстрый старт** (Git, Node.js, Claude Code уже есть): **[QUICK-START.md](docs/QUICK-START.md)** -- 15 минут до первой сессии.

**Полная установка** от чистого компьютера: **[SETUP-GUIDE.md](docs/SETUP-GUIDE.md)** -- 30-60 минут с установкой всех зависимостей.

```bash
mkdir -p ~/IWE && cd ~/IWE
gh repo fork TserenTserenov/FMT-exocortex-template --clone
cd FMT-exocortex-template
bash setup.sh
```

После установки:

```bash
cd ~/IWE
claude
```

Скажите Claude: **«Проведём первую стратегическую сессию»** -- и он проведёт вас через определение целей, создание первого плана и настройку среды.

## Кастомизация

IWE обновляется как дистрибутив — вы получаете обновления платформы, не теряя свои настройки.

**Расширения (extensions/)** — добавляйте свои блоки в протоколы:

```bash
# Добавить рефлексию в конце дня
echo "## Рефлексия дня
- Что было сложным?
- Что бы сделал иначе?
- За что похвалить?" > extensions/day-close.after.md
```

**Параметры (params.yaml)** — включайте/выключайте шаги протоколов:

```yaml
reflection_enabled: true    # Включить рефлексию
video_check: false          # Отключить проверку видео
multiplier_enabled: true    # Мультипликатор IWE
```

**Обновления** — `bash update.sh` обновляет платформу, сохраняя ваши extensions/, params.yaml и правки в CLAUDE.md (3-way merge).

Подробнее: [extensions/README.md](extensions/README.md)

## FAQ

**Q: Нужна ли подписка Anthropic?**
A: Для полной установки (Claude Code) — рекомендуется Claude Pro ($20/мес). При необходимости можно перейти на Claude Max (~$100/мес) для работы без ограничений. Для минимальной (`setup.sh --core`) — работает с любым AI CLI. Подробнее: [SETUP-GUIDE.md](docs/SETUP-GUIDE.md).

**Q: Работает ли с другими ИИ (не Claude)?**
A: Частично. Все знания хранятся в открытых форматах (Markdown, YAML, Git) -- они переживут смену любого вендора. Но автоматика (протоколы ОРЗ, скиллы, хуки, роли) заточена под Claude Code CLI. Для Codex (OpenAI), Aider или других AI CLI потребуется адаптация `.claude/` и скриптов ролей. Минимальная установка (`setup.sh --core`) работает без привязки к конкретному LLM.

**Q: Работает ли на Linux/Windows?**
A: Да. Ядро работает на любой ОС. Автоматизация Стратега: macOS — launchd, Linux — cron, Windows — WSL. Подробнее: [SETUP-GUIDE.md](docs/SETUP-GUIDE.md).

**Q: Что если компьютер выключен или спит — автоматика остановится?**
A: Cloud Scheduler (GitHub Actions) работает в облаке даже при выключенном компьютере. Для локальных агентов: скрипты автоматически предотвращают сон на время работы (macOS: `caffeinate`, Linux: `systemd-inhibit`). Для ноутбуков рекомендуется настроить автоматическое пробуждение и запрет idle sleep — см. [SETUP-GUIDE.md](docs/SETUP-GUIDE.md).

**Q: Что такое Pack?**
A: Формализованная область знаний — единственный source-of-truth для доменного знания. Подробнее: [LEARNING-PATH.md](docs/LEARNING-PATH.md).

**Q: Безопасны ли мои данные?**
A: Три зоны защиты: локальная, GitHub (приватные репо), платформа (per-user изоляция). Подробнее: [DATA-POLICY.md](docs/DATA-POLICY.md).

**Q: Чем IWE отличается от Obsidian / Notion / Logseq?**
A: Obsidian — хранилище заметок. IWE — **рабочая среда** с протоколами, ИИ-агентами и формализацией знаний. Вы можете использовать Obsidian внутри IWE для заметок, но IWE даёт структуру, планирование и накопление компетенций.

**Q: Нужно ли программировать?**
A: Нет. Шаблон — готовая конфигурация. Установка через setup.sh. Работа — через Claude Code на естественном языке.

**Q: Можно ли без Стратега?**
A: Да. Claude Code + CLAUDE.md + memory/ работают полностью. Стратег — автоматизация планирования. Без него планируете вручную.

**Q: Как настроить день стратегирования?**
A: В `memory/day-rhythm-config.yaml` измените `strategy_day: monday` на нужный день. Подробнее: [LEARNING-PATH.md](docs/LEARNING-PATH.md).

**Q: Клонирование попало в `~` вместо `~/IWE`?**
A: Все команды установки нужно выполнять в одном терминале. Если открыли новый — он начинает из `~`. Удалите папку из `~` и повторите с `cd ~/IWE`. Подробнее: [SETUP-GUIDE.md](docs/SETUP-GUIDE.md).

> Полный FAQ с подробными ответами: [DP.IWE.002 §11](https://github.com/TserenTserenov/PACK-digital-platform/blob/main/pack/digital-platform/02-domain-entities/DP.IWE.002-iwe-template-and-setup.md#11-частые-вопросы-faq). ## Сообщество IWE

IWE — это среда, которую строишь в одиночку. Но развиваешь — вместе.

**Сообщество IWE** — закрытый чат практиков, которые работают по той же системе: ОРЗ, Pack, экзокортекс. Место, где обсуждают не «как лучше промптить», а как строить интеллектуальную работу всерьёз.

### Что там происходит

- **Разборы рабочих продуктов** — участники показывают реальные Pack, планы, ретроспективы. Получают обратную связь от людей, которые понимают, что такое «закрытие без фиксации результата»
- **Опыт установки и кастомизации IWE** — что сломалось, как починили, какие расширения оказались полезными
- **Обсуждение методов** — ОРЗ-фрактал, АрхГейт, Capture-to-Pack на практике: что работает, где теория расходится с реальностью
- **Ссылки и находки** — инструменты, паттерны, SOTA, которые вписываются в философию IWE

### Почему это важно

Изучать систему в одиночку — можно. Но большинство вопросов возникают на этапе применения: «Как формализовать эту область знаний?», «Правильно ли я использую ОРЗ?», «У кого есть опыт с этим инструментом?»

В сообществе эти вопросы получают ответ от людей, которые через это уже прошли.
