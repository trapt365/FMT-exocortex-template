---
name: Критичная проверка транскриптов Deepgram на казахские реалии
description: Перед использованием аббревиатур и имён собственных из Deepgram-транскриптов прогонять их через здравый смысл (ЕНТ→EFT, ҰБТ, КТА, имена)
type: feedback
---

Перед тем как использовать в документах аббревиатуры, имена собственные
или специфические термины из Deepgram-транскриптов казахских разговоров —
**обязательно прогонять через здравый смысл** и контекст домена.

**Why:** В сессии 2026-04-07 я некритично перенёс «EFT» из аудио-транскрипта
в документ для Айдара (диагностические вопросы по образовательному
проекту i-School). Айдар поправил: «откуда термин EFT?». На самом деле
Deepgram распознал казахское «ЕНТ» (Единое национальное тестирование) как
«EFT». Это типичная ошибка ASR на казахских реалиях — звукоподобные
английские буквы вместо казахских аббревиатур.

**Известные паттерны искажений** (накапливать по мере встреч):
- ЕНТ → EFT, ENT, IFT
- ҰБТ → UBT, UBD
- КТА → KTA, CTA
- Казахские имена собственные часто искажаются (Нурлан → Нурланд, Айдар
  Рысбекулы → Айдар Рысбекуль)

**How to apply:**
1. Перед использованием **любой аббревиатуры из 2-4 латинских букв** в
   контексте казахского разговора — спросить себя: «это реальный
   английский термин или Deepgram распознал казахскую аббревиатуру?».
2. При сомнении — либо использовать наиболее вероятную казахскую
   аббревиатуру (ЕНТ для образования, ҰБТ для аттестации) с пометкой
   «*возможно Deepgram распознал X как Y, поправь если не так*»,
   либо пропустить термин и переформулировать.
3. Имена собственные (Нурлан, Айдар, …) — сверять с контекстом, не
   доверять написанию из транскрипта дословно.
4. Этот же принцип распространяется на любой ASR (Whisper, Soniox, …),
   когда речь идёт про казахские/русские реалии в смешанном контексте.
