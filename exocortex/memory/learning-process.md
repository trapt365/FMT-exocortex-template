## Inline-аннотации (3 маркера Obsidian)

При чтении подглав в clipping пользователь использует **три типа маркеров**. Семантика по типам не закреплена (пока не важно) — все три = **точки внимания**, которые R3 Консультант обязан отработать на ревью.

| Маркер | Синтаксис | Видимость | Типичное использование |
|--------|-----------|-----------|------------------------|
| Скрытый комментарий | `%%мой текст%%` | Только в edit mode | Вопросы, сомнения, мета-замечания |
| Footnote (inline) | `^[мой текст]` | В reading mode как сноска | Свои инсайты, операционализация, личные правила |
| Highlight | `==выделенный кусок==` | Желтый фон в reading | Ключевые цитаты автора, узнавание (риск false-completion!) |

**Роль R3 Консультант (правило `feedback-false-completion-learning.md`):**
- Все три типа = **первичные точки разбора**, не игнорировать ни один.
- На каждый `%%`/`^[...]`/`==` — встречный вопрос или просьба о примере применения.
- **Особое внимание к `==highlight==` без комментария:** highlight = узнавание ≠ применение. R3 обязан задать вопрос «как это применишь на боевом примере?», иначе подглава пройдена в режиме false-completion.

**Триггер разбора:** «разбери комментарии в R{X}.0{Y}» / «прошёл подглаву» → Claude читает clipping, отрабатывает каждый маркер по порядку.

## Anki-карточки (импорт в Anki приложение через TSV)

- **Формат:** **TSV для прямого импорта** в Anki приложение (File → Import). НЕ через Obsidian Flashcards плагин.
- **Файл карточек:** **отдельный файл на каждую подглаву** — `Vault1/{R{X}.0{Y}} Anki cloze.tsv` (пример: `R1.07 Anki cloze.tsv`). Если есть basic-карточки — отдельный файл `{R{X}.0{Y}} Anki basic.tsv`.
- **Заголовок TSV (метаданные Anki 2.1+):**
  ```
  #separator:tab
  #html:false
  #notetype:Cloze        (или Basic для basic-файлов)
  #deck:Default
  #tags column:3
  ```
- **Структура строки (Cloze):** `Text\tBack Extra\tTags` (3 поля через TAB; Tags — через пробел, например `r1-07 r1 firefighting`)
- **Структура строки (Basic):** `Front\tBack\tTags`
- **Cloze-синтаксис:** `{{c1::скрытое}}` / `{{c2::другое}}` (нативный формат Anki, НЕ `==выделение==` плагина)
- **Опциональный конспект:** рядом можно положить `{R{X}.0{Y}} Anki.md` — human-readable контекст карточек, не для импорта.
- **Импорт:** пользователь делает вручную (Anki приложение → File → Import → .tsv).
- **Форматы:**
  - Inline: `Вопрос :: Ответ`
  - Inline reverse: `Вопрос ::: Ответ`
  - Cloze: `==выделение==` или `{фигурные скобки}` (НЕ `{{c1::}}`)
  - Многострочная: `Вопрос #card` (тег в конце строки вопроса), ответ на следующих строках
- **Синхронизация:** плагин → AnkiConnect → Anki
