При Session Open учебной сессии по курсам Aisystant (СМ R1, R5, R6 и далее) — артефакт сессии = следующая подглава, **определяемая мной самостоятельно** из таксономии курса и текущей позиции. НЕ предлагать абстрактно «R1.2:1» из памяти WeekPlan и НЕ спрашивать «где ты остановился».

**Why:** 18 мая 2026 я при Session Open предложил R1.2:1 «Развитие» (из формулировки WeekPlan W20 «R1.1+R1.2»). Пользователь поправил: оглавление курса и текущая позиция у меня в memory загружены — нужно было самому определить следующую подглаву. По [[learning-aisystant-r6-progress]] на 15 мая R1.1 не закрыт (⬜ R1.1:12-14 + Tasks4-6, ⬜ R1.0:Mod0, R1.0:9), а DayPlan от утра 18 мая зафиксировал R1.2:1 — тоже моя ошибка при Day Open, не сверился с memory. Прыжок через незакрытые подглавы = нарушение таксономии R1 (последовательность важна, R1.1 даёт инструменты для R1.2).

**How to apply:**
1. При триггере учебной сессии («открываем СМ», «начинаем R1», «слот 1 учёбы») — ДО Ритуала прочитать [[learning-aisystant-r6-progress]] (или соответствующий learning-* для других курсов).
2. Найти первый ⬜ (непройденный) пункт в текущем разделе. Это и есть артефакт сессии. Если раздел закрыт (все ✅), брать первый ⬜ следующего раздела по нумерации курса.
3. В Ритуале объявить: *«Следующая подглава по [[learning-aisystant-r6-progress]] (last_updated YYYY-MM-DD) = R1.1:12 «<название>». Артефакт: подглава прочитана + R3-разбор + Anki.»* — утверждение, не вопрос.
4. Если DayPlan РП #1 указывает другую подглаву → сверить с memory, при расхождении предложить поправку DayPlan inline ([[feedback-dayplan-inline-update]]) и объяснить расхождение пользователю до старта работы.
5. Спрашивать только если: (а) memory не содержит конкретного ✅/⬜ для последнего пройденного, (б) пользователь явно сказал «хочу не следующую, а перепрыгнуть».
6. **Не путать с** [[feedback-learning-progress-source-of-truth]]: тот про Day Close (актуальный статус сегодня), этот — про Session Open (что делать дальше).
