**Правило:** при R3-разборе R1.1 (и шире R1.x до перехода в R5 СМ) **НЕ выносить категоричный вердикт** в формате «X = целевая система». Вместо этого — допускать описание заземлённой ситуации эксплуатации (конкретные физ. объекты + ограниченное время) без окончательного выделения системы.

**Why:** 15 мая в Слоте 1 (Tasks3 + R1.1:10 + R1.1:11) я (Claude) допустил эту ошибку дважды:

1. В разборе Tasks3 одобрил пользователю формулировку «целевая система = торговая операция в магазине» (8 мая) — на самом деле это WorkEpisode по R1.1:10.
2. В попытке поправить — переключился на категоричное «целевая система = магазин». Пользователь поймал дребезг при чтении R1.1:10, потому что глава прямо говорит:
   > «Если не получится подобраться к системе, ограничьтесь… Опишите конкретную, максимально заземлённую ситуацию эксплуатации… **Запомните, что вы пока не выделили систему**, и вернётесь к вашему рассуждению при прохождении резидентуры по моделированию [R5 СМ].»

То есть курс **легализует промежуточное состояние** «целевая система пока не выделена, но описана заземлённая ситуация». Категоричный вердикт R3-консультанта на этом этапе — это false-completion (фасадная окончательность).

R1.1:11 дополнительно подчёркивает: множественность точек зрения на «целевую систему» — норма, не дефект. Пример из R1.1:10 (нормоконтролёр): целевая система для создателя ≠ целевая система для университета.

**How to apply:**

При R3-сессии по R1.x:

1. **Категоричные формулировки «X = целевая система» — запрещены** до R5 СМ.
2. **Корректные ходы:**
   - «Кандидаты на целевую систему (по точкам зрения): …» — список с разными перспективами
   - «На этом этапе R1 разрешено описать заземлённую ситуацию: конкретный агент-Х в момент времени Y в физическом месте Z делает W»
   - «Окончательное выделение — в R5 СМ»
3. **Что ловить у пользователя** (false-completion гард):
   - «Целевая система = [отглагольное существительное]» — поймать по правилу 6 R1.1:2 / R1.1:10 (заморозка времени, фото vs видео)
   - «Целевая система = [множество агентов / собирательное имя типа клиентура]» — поймать по правилу 3 R1.1:2
   - «Целевая система = [описание / документ]» — поймать по правилу 7 R1.1:2
4. **Использовать заземлённую ситуацию как валидный итог R1-сессии** — без обязательной попытки её «свернуть» к одной целевой системе.

**Связанные различения:**
- Функция ≠ Система: «торговая операция» — функция магазина; магазин — кандидат на физическую систему
- Агент = специальная система-создатель (R1.1:11); FPF: «агент» → «U.System + RoleAssignment»
- WorkEpisode (FPF) — событие смены состояний агентов; не путать с самой системой

**Применимо также к:** обсуждению РП с пользователем — там та же ошибка может приводить к псевдо-конкретике «делаем систему X», когда X — это процесс, а не 4D-объект.
