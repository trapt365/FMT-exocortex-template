---
name: Открытие файлов в терминале
description: Пользователь работает из non-integrated WSL-терминала — кликабельные пути не работают, открывать через `code <abs-path>`
type: feedback
---

Когда нужно дать пользователю ссылку на файл — НЕ полагаться на кликабельность пути в терминале. Вместо этого выполнять `code <абсолютный путь>` через Bash, чтобы файл открылся во вкладке VS Code.

**Why:** Пользователь запускает Claude Code из обычного WSL-терминала Ubuntu (не из VS Code Integrated Terminal в WSL Remote режиме). Диагностика 2026-04-07: `TERM_PROGRAM` пусто, `VSCODE_IPC_HOOK_CLI` не установлена, `code` = `/mnt/c/Program Files/Microsoft VS Code/bin/code`. В таком терминале нет линк-детектора — ни относительные, ни абсолютные пути, ни `file://` URI не становятся кликабельными. Это не баг VS Code, это архитектура: кликабельность путей реализована в renderer'е VS Code Integrated Terminal, а не в conhost/Windows Terminal.

**How to apply:**
- При создании/изменении файла, к которому пользователь захочет вернуться → сразу после Write/Edit вызывать `code <абсолютный путь>` (если файл важный для следующего шага пользователя)
- Если пользователь говорит «открой», «покажи», «давай посмотрим на файл» → `code <abs-path>`, не «вот путь, открой»
- Абсолютные пути в ответе всё равно давать (для копирования, для контекста), но не как «кликабельную ссылку»
- Долгосрочный фикс для пользователя: запускать Claude Code из VS Code Integrated Terminal в WSL Remote режиме (`code /home/trapt22/IWE` → `Ctrl+\`` → `claude`). Тогда `TERM_PROGRAM=vscode` и Ctrl+Click заработает естественно. Но пока пользователь не переехал — использовать workaround выше.
