## Формула (5 шагов)

1. **SCP** (замечания клиента/стейкхолдеров)
2. **Code Review** (3-layer adversarial) — после КАЖДОГО раунда крупных правок, не только после SCP
3. **Elicitation** (BMad, параллельно где можно):
   - Shark Tank (adversarial — неявные допущения)
   - Stakeholder Round Table (audience — пробелы для читателей)
   - → Correct Course (фильтр — что реально в scope)
   - → Реализация одобренных
   - → Self-Consistency (что сломалось) — после КАЖДОГО раунда правок
   - Comparative Matrix, Feynman, Critique & Refine — по необходимости
4. **Финальный коммит**

**Why:** Выведено из propresearch v1.0→v1.4.1 (~40 находок за 7 раундов). Без Correct Course документ становится спекулятивным. Без Self-Consistency после каждого раунда — 5+ рассинхронов.

**How to apply:** Минимум 3 метода на документ: adversarial + audience-based + consistency.

## Каскадный чеклист

После каждой правки в DR проверить: DR → ES → dependency map → changelog → glossary. Версию бампить ОДИН РАЗ за сессию.

## Принцип бесплатных данных

Если данные доступны бесплатно и быстро (AI-парсинг) — включай в Phase 1, даже если формально вне scope. Увеличивает value без увеличения стоимости.

**Why:** Парсинг коливинга (krisha.kz) = 0 тг, 30 мин. Не включили в v1.3 («не в scope»). Клиент потом попросил — сделали за 30 мин то, что могли сделать сразу.

## Локализация

Локализацию терминов делать ДО глоссария. Формат: «Русский термин (English original)». replace_all по терминам — проверять глоссарий отдельно (ломает скобки).
