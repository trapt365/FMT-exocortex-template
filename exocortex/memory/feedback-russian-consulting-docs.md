В консалтинговых, методологических и стратегических документах (КП, memo, scope, methodology-rethink, feasibility, презентации Айдару/клиентам) — придерживаться русского, не злоупотреблять англицизмами. Расшифровывать аббревиатуры при первом упоминании.

**Why:** Документ должен читаться без знания индустрии (Айдар + клиент-заказчик в KZ — генподрядчик и владелец проекта, не hospitality-аналитики). Аббревиатуры (HVS, ULI, JLL, STR, ADR, RevPAR, USALI, DCF, NPV, IRR, UHNW, MPI/ARI/RGI) и калька (pipeline, output, scope, framework, baseline, deliverables, sub-supplier, compset, validation, bankable) — создают барьер. Дублируется правило distinctions.md «Ru-first SPF §5 п.13», но пока я регулярно нарушаю — нужно явное напоминание.

**How to apply:**

- **Имена собственные** (HVS, ULI, JLL, Accor, Swissôtel, Cushman & Wakefield, Knight Frank, Savills, Cornell CHR, STR/CoStar) — оставить как есть, но при первом упоминании в документе дать короткое пояснение в скобках или в сноске.
- **Аббревиатуры терминов** — расшифровать при первом упоминании:
  - ADR → средняя цена номера (Average Daily Rate)
  - RevPAR → выручка на номер в наличии
  - USALI → стандарт гостиничной отчётности
  - DCF → дисконтированный денежный поток
  - NPV / IRR → чистая приведённая стоимость / внутренняя норма доходности
  - MPI / ARI / RGI → индексы проникновения (по загрузке / по цене / по выручке)
  - UHNW / VHNW → сверхбогатые / очень богатые
  - HOA → товарищество собственников
- **Кальки заменять русскими:**
  - pipeline → последовательность шагов / конвейер
  - output → результат
  - input → исходные данные
  - scope → объём работ
  - framework → методика / каркас
  - baseline → опорные данные / базовая линия
  - deliverables → результаты работы / итоговые документы
  - sub-supplier → субподрядчик
  - compset → конкурентная выборка / референтная группа
  - validation → проверка / валидация (если без альтернативы)
  - bankable / non-bankable → пригодное / непригодное для банковского финансирования
  - quote → расчёт / коммерческое предложение
- **Допустимо оставлять английский:** YAML-ключи, имена брендов, названия методик (Trend Report, Wealth Report), технические идентификаторы (B967, USALI 11/12).

**Применять к:** WP-hotelresearch (текущая работа), WP-13 propresearch, WP-mebelresearch, любой будущий research-консалтинг для внешнего заказчика, презентации.

**Не применять к:** внутренним IWE-документам (memory, CLAUDE.md, hooks, скрипты) — там аббревиатуры допустимы. Триггер — «документ для внешнего получателя» (клиент / партнёр / lawyer).
