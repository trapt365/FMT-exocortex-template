# Day Close спринт-чеклист — задавать вопрос, не писать заглушку

**Правило:** на шаге «Спринт-чеклист (бинарные метрики)» в Day Close — **ВСЕГДА задать пользователю один вопрос-блок** со списком метрик. Заглушка вида «Бинарные метрики: F1-1, F1-2... — секция дополняется при ответе» = **нарушение протокола**, считать пропущенным шагом.

**Why:** Day Close 2026-05-01 ([archive/day-plans/DayPlan 2026-05-01.md:281](DS-strategy/archive/day-plans/DayPlan%202026-05-01.md#L281)) написал заглушку вместо вопроса → данные по 9 метрикам (F1-1, F1-2, F1-4, F2-2, К1-1, К2-1, К2-2, Л2-1, Л2-3) **потерялись**. Корней два:
1. **Структурный:** `load-extensions.sh` отсутствовал в IWE-корне → extension `day-close.after.md` не загрузился → агент не получил инструкцию из шага 2 («Спросить пользователя одним вопросом»). Закрыто 2026-05-02 wrapper'ом в `.claude/scripts/load-extensions.sh` (см. [bug-2026-05-02-load-extensions-missing.md](DS-strategy/inbox/bugs/bug-2026-05-02-load-extensions-missing.md)).
2. **Поведенческий:** даже без extension агент видел секцию-заголовок «Спринт-чеклист» с автоопределяемыми метриками. На бинарных вместо вопроса написал «нужен ответ пользователя» — это нарушение «Choice-question» исключения в Правиле 6 (Автономность). Choice-вопросы (done/skip по 10 метрикам) — нормальный режим collaboration, **обязаны задаваться**.

**How to apply:**

**Триггер:** Day Close, секция «Спринт-чеклист (D{N})» — после автоопределяемых метрик из коммитов/DayPlan.

**Что делать:**
1. Прочитать [extensions/day-close.after.md](extensions/day-close.after.md) (через wrapper `load-extensions.sh`).
2. Из таблицы метрик отделить «спросить» от «DayPlan/коммиты» (10-12 «спросить»).
3. **Задать ОДИН вопрос-блок** в формате:
   > *«Спринт-чеклист: (1) без экранов? (2) клетчатка? (3) магний? (4) тебе-экспрессия? (5) 4к шагов? (6) пианино? (7) шахматы? (8) гребля Z2? (9) силовая? (10) Anki?»*
4. **Дождаться ответа** пользователя (одна строка вида «1 done, 2 skip, 3 done, ...» или ad-hoc).
5. Записать `done`/`skip` по каждой строке в секцию `### Спринт (DD мес)` DayPlan.
6. Если slim-режим (F2-1 + F2-2 PAUSE) — снять соответствующие коды из вопроса, остальные задавать.
7. Если пользователь явно сказал «пропусти чеклист» / «без вопросов» — записать «спринт-чеклист пропущен по запросу».

**Запрещено:**
- Заглушка «секция дополняется при ответе» / «нужен ответ пользователя» / «спросить позже» — это пропуск шага, не его выполнение.
- Угадывать значения метрик из косвенных признаков («наверное, делал клетчатку, потому что был дома»).
- Задавать вопросы по одной метрике (10 раундов вопрос-ответ) — только один блок.

**Связанные правила:** Правило 6 (Автономность) исключение «Choice-question» — выбор done/skip по 10 метрикам это choice-режим. Не путать с yes/no согласием на готовое решение, которое запрещено.
